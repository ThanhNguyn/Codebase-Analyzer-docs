Overview

Codebase Analyzer

Công cụ phân tích mã nguồn cục bộ — kiến trúc OOP với C++23 core engine và giao diện desktop Electron + React. Bài tập lớn môn Lập Trình Hướng Đối Tượng.

Mục tiêu dự án

Xây dựng Codebase Analyzer dưới hai chế độ: Desktop UI App và CLI Mode. Cho phép người dùng chọn thư mục mã nguồn, quét đệ quy toàn bộ cây thư mục, tự động bỏ qua các folder nhiễu, nhận diện file source và thống kê các chỉ số cốt lõi gồm tổng số file, dòng code, dòng comment và dòng trống.

Đối tượng người dùng

Sinh viên ngành CNTT, kỹ sư phần mềm, quản lý dự án — bất kỳ ai cần đánh giá, kiểm tra cấu trúc và quy mô của một kho lưu trữ mã nguồn cục bộ (Local Codebase).

Quét thư mục
Duyệt đệ quy thư mục, tự động phát hiện file mã nguồn hợp lệ, bỏ qua thư mục noise.
Phân tích ngôn ngữ
Nhận diện ngôn ngữ theo extension, áp dụng parsing riêng biệt cho từng loại.
Đếm comment
Đếm single-line & multi-line comments theo cú pháp từng ngôn ngữ lập trình.
Xuất báo cáo
Tạo báo cáo Markdown với thống kê chi tiết từng file và ngôn ngữ.
Architecture

System Architecture

Chia thành 2 layer chính: C++ CLI Core xử lý phân tích và Desktop UI hiển thị kết quả.

C++ CLI Core

DirectoryScanner — Quét đệ quy, lọc thư mục noise (.git, node_modules, build...), hỗ trợ .gitignore rules

FileAnalyzer (Abstract) — Base class với pure virtual analyze(), languageName(), language()

Language Analyzers — CppAnalyzer, CAnalyzer, PythonAnalyzer, JavaAnalyzer, CSharpAnalyzer, JavaScriptAnalyzer, TypeScriptAnalyzer, WebAnalyzer, HtmlAnalyzer, CssAnalyzer

ReportGenerator — Xuất console + Markdown report

Desktop UI

Electron — Main process, window management, file system access

React + TypeScript — Frontend components, type safety

Vite — Build tool, Hot Module Replacement

Tailwind CSS — Styling framework

OOP Design

OOP Principles

Minh họa đầy đủ 4 nguyên lý cốt lõi của Object-Oriented Programming.

Abstraction
FileAnalyzer định nghĩa interface chung cho tất cả analyzer. Các phương thức analyze(), languageName(), language() là pure virtual.
virtual void analyze() = 0;
Inheritance
Các analyzer cụ thể (CppAnalyzer, PythonAnalyzer, ...) kế thừa từ FileAnalyzer, tái sử dụng logic chung và mở rộng riêng.
class CppAnalyzer : public FileAnalyzer { ... };
Polymorphism
analyze() được gọi thông qua con trỏ base class (unique_ptr<FileAnalyzer>), runtime quyết định hàm nào được thực thi.
vector<unique_ptr<FileAnalyzer>> files;
Encapsulation
Dữ liệu (codeLines_, commentLines_, blankLines_) là protected. Truy cập qua getter với [[nodiscard]] attribute.
[[nodiscard]] std::size_t codeLines() const noexcept;

Tech Stack

Tech Stack

C++23
std::filesystem, std::unique_ptr, STL containers, RAII pattern, [[nodiscard]].
CMake 3.20+
Cross-platform build system. Hỗ trợ Windows (MSVC/MinGW), Linux (GCC), macOS.
Electron
Desktop app framework. Main process quản lý window và file system access.
React + TypeScript
Frontend UI components với TypeScript type safety.
Vite
Build tool với Hot Module Replacement cho development nhanh.
GitHub Actions
CI/CD pipeline, tự động build release cho multi-platform.

Core Libraries

Các thư viện chuẩn C++ được sử dụng trong CLI core engine.

<filesystem>
Sử dụng recursive_directory_iterator để quét cây thư mục an toàn, đa nền tảng và nhận diện phần mở rộng tệp tin nhanh chóng.
<memory>
Áp dụng std::unique_ptr để quản lý vòng đời mảng đa hình, tự động giải phóng bộ nhớ và ngăn ngừa hoàn toàn Memory Leak.
<string> & <algorithm>
Xử lý chuỗi tối ưu bằng STL thay vì Regex — tăng tốc I/O gấp nhiều lần khi phân tích các dự án lớn.
